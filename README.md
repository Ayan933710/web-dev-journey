# Web Development Journey

Welcome to my full-stack web development knowledge base! This repository serves as a chronological archive of my growth, core conceptual learning, and feature experimentations as a developer. It tracks my structured progression from learning the foundational building blocks of the web to developing complete server-side logic and database models.

---

## 📂 Repository Structure

The code in this repository is strictly organized by technology stack and learning modules:

### 🎨 1. Frontend

Focused on user interface architecture, semantic structures, and responsive design systems.

- `HTML/` - Core semantic layouts, accessibility standards, and web document structures.
- `CSS/` - Layout styling, box-modelling, flexbox/grid architectures, and animations.
- `JavaScript/` - ES6+ syntax, asynchronous programming, DOM manipulation, and logic execution.
- `BootStrap/` - Component-based rapid UI prototyping and responsive grid frameworks.

### ⚙️ 2. Backend

Transitioning from client-side interfaces to secure, robust server-side development.

- `Node/` - Server-side JavaScript runtime environments, event loops, and package management.
- `Express/` - Building scalable APIs, implementing middleware, and structuring routing logic.
- `REST/` - Designing architectural patterns for clean, standard HTTP CRUD endpoints.
- `EJS/` - Server-side rendering, layout templating, and injecting dynamic server data into views.

### 🗄️ 3. Database

Data modeling, system architecture, and persistent storage management.

- `MongoDB/` - Document-based NoSQL database management, schema validation, and aggregation.
- `SQL/` - Relational database systems, structured query design, data constraints, and optimizations.

### 🛠️ 4. Minor Projects

A sandbox of smaller, high-velocity mini-projects built to practically apply specific technical concepts, including:

- **Feature Experiments:** Isolated builds testing new layouts, styling practices, or API logic.
- **Quick Applications:** Functional tools (e.g., the `To_Do_List` app) testing DOM manipulation and client-side logic.
- **Proof of Concepts:** Smaller mockups built to understand a technology before scaling it into standalone systems.
